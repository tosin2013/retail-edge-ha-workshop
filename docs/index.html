Workshop Overview

This hands-on lab teaches you to design, deploy, and test high-availability systems for heavily constrained retail storefronts using OpenShift Virtualization and User Defined Networks (UDNs).

4 Lab Modules
50 Max Students
9 VMs per Student
3 HA Architectures

Architecture

The workshop uses a GitOps-based approach with Helm App of Apps pattern:

ArgoCD Parent Application (retail-edge-ha-workshop)
│
├── Wave 0: Infrastructure
│   ├── Student Namespaces (retail-edge-student-01..50)
│   ├── UDN Namespaces (retail-edge-student-01-udn..50-udn)
│   ├── Resource Quotas (CPU: 16, Memory: 32Gi, Storage: 200Gi)
│   └── Operator Configurations
│
├── Wave 1: Networking & RBAC
│   ├── User Defined Networks (UDNs)
│   │   ├── Module 1: pacemaker-net (10.101.0.0/24)
│   │   ├── Module 2: microshift-net (10.102.0.0/24)
│   │   └── Module 3: twonode-net (10.103.0.0/24)
│   ├── NetworkAttachmentDefinitions
│   ├── ClusterRoles
│   └── RoleBindings
│
├── Wave 2: VirtualMachines
│   ├── Module 1: RHEL HA VMs (2 per student)
│   ├── Module 2: MicroShift VMs (2 per student)
│   └── Module 3: Two-Node OpenShift VMs (3 per student)
│
└── Wave 3: Bookbag
    ├── Workshop Content Deployment
    ├── Embedded Web Terminal
    └── Route for Student Access

Key Technologies

  • OpenShift 4.21+ - Container platform with Virtualization
  • User Defined Networks (UDNs) - Layer 2 networking for VM-to-VM communication
  • KubeVirt - Virtual machine management on Kubernetes
  • ArgoCD - GitOps continuous delivery
  • Helm - Package manager for Kubernetes
  • Bookbag - Containerized workshop content delivery

Architecture Decision Records

  • ADR-0001: Helm-based App of Apps Pattern Accepted

    Why we chose Helm with ArgoCD App of Apps for declarative, scalable multi-user deployment.

  • ADR-0002: Multi-User Namespace Isolation Strategy Accepted

    Dual namespace pattern (workload + UDN) for security isolation and resource fairness.

  • ADR-0003: User Defined Networks for Layer 2 VM Connectivity Accepted

    Using OVN-Kubernetes UDNs to provide true Layer 2 networking for VRRP and Corosync.

  • ADR-0004: Bookbag for Workshop Content Delivery Accepted

    Containerized lab guide with embedded web terminal and personalized student variables.

  • ADR-0005: ArgoCD Sync Wave Strategy Accepted

    Ordered deployment using sync waves (0=infra, 1=network, 2=VMs, 3=apps).

Quick Start Guide

Prerequisites

  • OpenShift 4.21+ cluster
  • OpenShift GitOps (ArgoCD) operator installed
  • OpenShift Virtualization operator installed
  • oc and virtctl CLI tools

1. Login to OpenShift

oc login --token=<your-token> \
  --server=https://api.cluster-ntq88.dynamic.redhatworkshops.io:6443

2. Deploy Workshop via ArgoCD

oc apply -f - <<EOF
apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: retail-edge-ha-workshop
  namespace: openshift-gitops
  finalizers:
    - resources-finalizer.argocd.argoproj.io
spec:
  project: default
  source:
    repoURL: https://github.com/tosin2013/retail-edge-ha-workshop.git
    targetRevision: main
    path: helm/retail-edge-ha
    helm:
      valueFiles:
        - values.yaml
  destination:
    server: https://kubernetes.default.svc
    namespace: retail-edge-infrastructure
  syncPolicy:
    automated:
      prune: true
      selfHeal: true
    syncOptions:
      - CreateNamespace=true
      - ServerSideApply=true
EOF

3. Wait for Deployment

# Watch ArgoCD sync
argocd app wait retail-edge-ha-workshop --health --timeout 600

# Verify components
oc get namespaces -l workshop=retail-edge-ha
oc get userdefinednetworks --all-namespaces
oc get vms --all-namespaces -l app.kubernetes.io/part-of=retail-edge-ha

4. Access Workshop

# Get Bookbag route
BOOKBAG_URL=$(oc get route bookbag -n retail-edge-bookbag -o jsonpath='{.spec.host}')
echo "Workshop URL: https://${BOOKBAG_URL}"

Configuration

All workshop settings are controlled via helm/retail-edge-ha/values.yaml:

# Student count (1-50)
students:
  count: 5

# OpenShift cluster settings
global:
  clusterDomain: apps.cluster-ntq88.dynamic.redhatworkshops.io
  clusterApiUrl: https://api.cluster-ntq88.dynamic.redhatworkshops.io:6443

# Enable/disable modules
virtualMachines:
  module1:
    enabled: true  # RHEL HA
  module2:
    enabled: true  # MicroShift
  module3:
    enabled: true  # Two-Node OpenShift

# Resource allocations per student
resourceQuotas:
  limits:
    cpu: "16"
    memory: "32Gi"
    requests.storage: "200Gi"
