== Key Lessons Learned

=== Pacemaker/Corosync

* STONITH fencing is critical for data integrity
* fence_kubevirt integrates seamlessly with OpenShift Virtualization
* Two-node clusters require `no-quorum-policy=ignore`
* Virtual IPs provide service continuity during failover

=== MicroShift with VRRP

* Keepalived is lightweight and battle-tested
* Health checks enable application-aware failover
* VRRP multicast requires Layer 2 connectivity
* Preemption ensures VIP returns to preferred node

=== Two-Node OpenShift

* etcd quorum math: ⌊N/2⌋ + 1 (for N=3, quorum=2)
* Remote arbiter minimizes edge resource requirements
* Bootstrap-in-Place simplifies installation
* Network latency is critical for etcd performance

=== Chaos Engineering

* Deliberate failure testing builds confidence
* Automated recovery is faster than human intervention
* Monitoring and alerting are essential for invisible failures
* Every failure is a learning opportunity

== Production Readiness Checklist

Before deploying these architectures in production:

=== Pacemaker

- [ ] Configure monitoring (pcs status, resource health)
- [ ] Set up alerting for cluster state changes
- [ ] Test STONITH fencing in production environment
- [ ] Document recovery procedures
- [ ] Implement backup/restore for cluster configuration

=== MicroShift VRRP

- [ ] Adjust VRRP priority based on hardware capability
- [ ] Configure application-specific health checks
- [ ] Monitor VRRP state transitions
- [ ] Test VIP migration during maintenance windows
- [ ] Document firewall requirements (VRRP protocol)

=== Two-Node OpenShift

- [ ] Measure network latency to arbiter (<10ms RTT)
- [ ] Use SSD/NVMe storage for etcd
- [ ] Monitor etcd metrics (consensus time, disk sync)
- [ ] Plan update/upgrade strategy (test extensively)
- [ ] Configure remote arbiter with redundancy
