= Welcome to the Retail Edge HA Workshop

== Introduction

Welcome! In this hands-on workshop, you'll learn to deploy and manage three different high-availability architectures commonly used in retail edge computing environments. Your lab VMs are managed by **Red Hat Edge Manager** (Modules 1 and 2) and **Red Hat Advanced Cluster Management** (Module 3), giving you hands-on experience with production fleet management tools.

== Workshop Overview

Retail edge locations face unique challenges:

* Limited physical space and resources
* Potential for hardware failures
* Network connectivity issues
* Need for continuous operation

This workshop demonstrates three production-ready HA patterns that address these challenges.

== What You'll Learn

Over the next few hours, you'll work through five modules:

[cols="1,3"]
|===
|Module |Description

|*Module 0*
|Explore fleet management with Red Hat Edge Manager and Red Hat Advanced Cluster Management (RHACM)

|*Module 1*
|Deploy a two-node Pacemaker/Corosync cluster with STONITH fencing + shared storage (GFS2). VMs managed by Edge Manager.

|*Module 2*
|Configure MicroShift with VRRP-based virtual IP failover + understand local storage limitations. VMs managed by Edge Manager.

|*Module 3*
|Install a two-node OpenShift cluster with remote etcd arbiter + learn etcd storage requirements. Cluster managed by RHACM.

|*Module 4*
|Perform chaos engineering tests to validate resilience
|===

NOTE: Each module includes a storage resilience exercise to help you understand how data persistence works in different HA architectures.

== Tools You'll Use

Throughout this workshop, you'll use:

* `oc` - OpenShift CLI (available in terminal)
* `virtctl` - KubeVirt VM management (available in terminal)
* `flightctl` - Red Hat Edge Manager CLI (available in terminal)
* `pcs` - Pacemaker cluster management (installed in VMs)
* `keepalived` - VRRP daemon (installed in VMs)
* `etcdctl` - etcd cluster inspection (installed in VMs)
