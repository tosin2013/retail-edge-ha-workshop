= ocp4_workload_retail_edge_ha - Retail Edge HA Workshop

== Role Overview

This AgnosticD workload role deploys the *Retail Edge HA Workshop* to an existing OpenShift 4.21+ cluster with OpenShift Virtualization. The workshop teaches students three different high-availability architectures commonly used in retail edge computing environments:

* *Module 1*: Two-node Pacemaker/Corosync cluster with STONITH fencing
* *Module 2*: MicroShift with VRRP-based virtual IP failover
* *Module 3*: Two-node OpenShift cluster with remote etcd arbiter

The role uses a *wrapper pattern* that orchestrates the existing Helm chart deployment rather than reimplementing resources in Ansible. This approach preserves the proven GitOps architecture while integrating with AgnosticD's lifecycle management.

== Architecture

The workshop deploys using:

* *GitOps*: ArgoCD App of Apps pattern (1 parent + 8 child applications)
* *Multi-user Isolation*: Dual namespaces per student (workload + UDN)
* *Networking*: User Defined Networks (UDNs) for Layer 2 connectivity
* *VMs*: 3-9 VirtualMachines per student (depending on modules enabled)
* *Lab Guide*: Showroom instances with embedded terminals

=== Resource Requirements

Per student (all modules enabled):

* *CPU*: 18 cores (2+2+2 for Module 1-2, 4+4+1 for Module 3)
* *Memory*: 36 GiB (4+4+4 for Module 1-2, 16+16+2 for Module 3)
* *Storage*: 200 GiB
* *Namespaces*: 2 (workload + UDN)

For 50 students (max): ~900 CPU cores, 1.8 TiB RAM, 10 TiB storage

== Prerequisites

=== Required Cluster Configuration

* OpenShift 4.21 or later
* OpenShift Virtualization operator installed
* Storage class with RBD support (or default storage class)
* OVN-Kubernetes CNI (for User Defined Networks)
* ArgoCD operator (installed by Helm chart if not present)

=== Required Tools in Execution Environment

* Helm 3.x (included in AgnosticD v2 EE)
* Git (included in AgnosticD v2 EE)
* kubernetes.core Ansible collection

== Cleanup

=== Remove Workshop

[source,bash]
----
./bin/agd destroy \
  --guid workshop-test \
  --config openshift-workloads \
  --account sandbox1234
----

This will:

1. Uninstall the Helm release
2. Delete ArgoCD applications (cascades to all child resources)
3. Remove student namespaces (VMs, UDNs, quotas)
4. Clean up GitOps namespace

== Troubleshooting

=== Common Issues

==== Issue: Virtualization Operator Not Found

*Error*: `OpenShift Virtualization operator must be installed and ready`

*Solution*:
[source,bash]
----
# Install OpenShift Virtualization operator
oc create -f https://raw.githubusercontent.com/kubevirt/hyperconverged-cluster-operator/main/deploy/deploy.sh
----

==== Issue: Storage Class Not Available

*Error*: `No storage classes found on cluster`

*Solution*: Ensure at least one storage class exists. The role auto-selects RBD storage if available, otherwise uses the default storage class.

==== Issue: ArgoCD Application Not Syncing

*Symptom*: ArgoCD application stuck in "OutOfSync" state

*Solution*:
[source,bash]
----
# Force sync the parent application
oc patch application retail-edge-ha-parent \
  -n retail-edge-ha-gitops \
  --type merge \
  -p '{"operation":{"initiatedBy":{"automated":true}}}'
----

==== Issue: VirtualMachines Not Starting

*Symptom*: VMs created but not running

*Solution*: Check if `autoStart` is enabled in values or manually start VMs:
[source,bash]
----
virtctl start rhel-ha-node1 -n retail-edge-student-01
----

== Integration with RHPDS

This role is designed for integration with Red Hat Product Demo System (RHPDS). Once tested, it can be:

1. Added to AgnosticD v2 core workloads collection
2. Listed in RHPDS catalog as `ocp4_workload_retail_edge_ha`
3. Auto-provisioned for workshops with OpenShift Virtualization clusters

== Development and Testing

=== Local Testing

[source,bash]
----
# Clone workshop repository
git clone https://github.com/tosin2013/retail-edge-ha-workshop.git

  --account sandbox1234 \
